  2. go build 编译
  3. package 与 import
  4. 命令行参数
    • os.Args 获取命令行参数
  5. 变量声明
    • var 变量名字 类型 = 表达式
    • 简短声明的方式 名字 := 表达式
    • 类型声明 type 类型名字 底层类型
    • new 关键字声明得到的是指针
    • 指针即为 *类型 指针可比较
  6. range 的用法 foreach 返回两个值 一个是索引 一个是值的内容
  7. 关于占位符，对于返回多个值的函数或者是谓词，可以忽略后面的占位符
    • 并不是 返回多个值的函数所有值都需要表示
  8. string记录的是UTF8编码的数组 bytes.Buffer动态增长的字符串类型
  9. 常量声明 const 名字 基础类型名字
    • 常量是无类型的 能够直接赋值变量 进行自动转换 转换的前提是目标类型能够容纳原常量
    • 无类型的常量 默认为int float64 complex128
  10. 数组 固定长度 特定类型 [num]type 数组加数组长度为一个复合类型
  11. slice 变长序列 []T len slice的长度 cap slice的容量 为数组的引用 name := array[begin:end] 为引用
    • nil的slice没有底层数组 len 和 cap 为 0 但是len和cap为0的有可能不是nil，但是他们处理方式相同
  12. map[K]V 向一个空的map插入元素会异常 其他空map操作正常
  13. 结构体 type name struct{} 能够直接通过指针以点操作符访问成员 成员名的首字母大写即为可导出的
    • 匿名成员 结构体的成员为一个命名结构体时可以省略名字
  14. 函数
    • 声明 func name(parameter-list) (result-list) {}
    • 错误
      • go将函数运行失败作为预期的返回值返回 主要有一下的返回值
      • error 接口
      • ok boolean
      • 错误处理方式：直接与处理后返回 重试 输出错误信息返回 输出错误信息不返回
    • 函数值为一种引用类型 不可比较 能够当做函数参数 作为一种状态，将自己的局部状态作为值
    • 匿名函数 函数的字面值语法，与函数声明相同，但是没有函数名 作为一种闭包的形式执行，注意闭包中变量的引用问题 range循环中 和 for循环中 go语句和defer语句
    • 可变参数 …类型 切片传入 slice_name…
    • 谓词defer 在函数返回时执行语句 和java不同的是defer能够修改返回值
    • 谓词panic 触发异常主要用于导致程序崩溃的错误，上层也无法解决的错误，程序只能崩溃 panic的参数为异常信息通常为接口
    • 谓词recover 获取panic异常的相关信息 处理该异常
  15. 面向对象与方法
    • 方法选择器 func (name structname) funcName(para-list) (re-list){}
    • 方法选择器不能更新对象 只有方法选择器中的对象为指针时才能更新对象 go有自动取地址的功能
    • 方法选择器返回的是一个函数值
    • 方法表达式 即通过类型返回的方法可以作为普通函数使用，第一个参数作为接收器
    • 封装的方式 通过 大小写字母实现
  16. 面向对象与结构
    • 调用方和实现方的约定致使调用可替换
    • type name interface{} 接口描述了一系列方法的集合 接口里的方法定义不需要加func
    • 空接口类型 interface{}
    • 接口可以实现在指针上，但是这样不能进行转换
    • 变量分为 类型 和 值 空接口值的变量调用方法会报错 动态分配给接口实际的类型
    • 接口可比较 接口比较的规则是nil或者是相同的可比较的类型来比较接口值 接口只有类型和值都为nil时才为nil
    • 常用的接口
      • sort.Interface
      • error Error() string
    • 类型断言 x.(T) 对nil进行断言会panic 返回一个值时表示是否为这个值 返回两个值时 第一个表示这个类型的值，第二个表示是否是这个类型的值（否的话第一个值会是nil）
    • 类型开关 接口的作用 一个是实现方法而不用专注具体的类型，一个是调用者只需要关心接口的类型，调用完成逻辑 switch x.(type) 判断类型 类型开关也会返回新的x值
  17. go并发
    • goroutine main运行在一个goroutine go谓词新建一个goruntine 新的函数会在goruntine中执行
    • channels 数据通信的类型 chan type 为引用
      • 发送 chan <- var 接受 <- chan
      • 可关闭 关闭后不可发送 可接受 close内建函数
      • 缓存限制 make(chan type, int)
      • 无缓存 同步channels 阻塞 routine happens before
      • chan接收时可以选择第二个参数 表明chan是否关闭 可是用range谓词遍历chan 可运行知道chan关闭 chan不需要显式的关闭，在没有引用时会被自动回收 重复关闭chan异常
      • 单向chan chan<-type 只发送 <-chan type 只接受 只有发送者才能close(chan)
      • 缓存chan cap(chan) 空间 len(ch) 长度
      • 泄露的goroutinue不会被回收
      • sync.WaitGroup() Add() Done() Wait() 能够实现计数等待的功能
      • select语句 select{case …} 监听多个channel的发送与接收 在某个chan有事件发生时就执行该分支
      • channel的零值为nil，nil的channel发送与接收操作永远阻塞
      • 通过关闭channel实现广播
  18. 基于共享变量的并发
    • 数据竞争 两个以上的goroutine并发访问相同的变量且其中一个至少为写操作时
    • 避免数据竞争的方式
      • 不进行写操作
      • 避免通过多个goroutine访问变量 不使用共享数据通信，使用通信共享数据 以channel实现
      • 互斥
    • sync.Mutex Lock Unlock 通过监控保证变量被顺序访问 封装能够使我们获得数据结构的不变性，mutex能够实现并发的不变性（并发安全）
    • sync.RWMutex 多读单写 RLock() RUnlock()
    • 单个goroutine的执行顺序是确定的
    • sync.Once 保证单例模式的实现
    • 竞争条件检测 -race
    • 基于通信机制实现并发程序
    • goroutine和线程
      • 线程是固定栈 goroutine动态栈
      • 线程上下文切换由内核完成 goroutine由go完成
      • goroutine没有线程局部存储
  19. 包和工具
    • 包的编译：导入包开头声明 禁止包环依赖 目标文件记录包的依赖关系
    • 包的导入：每个包由全局唯一的字符串所标识的导入路径定位
    • 包名 最后一段
    • 包的导入 可以给包重命名
    • 工具
      • go get 获取的代码是真实的仓库 -u 最新
      • go env
      • go build 相对路径 绝对路径 当前路径所有包 不加路径为GoPath环境变量下的路径 -i安装每个目标依赖的包
      • go run 直接编译后运行包
      • go install 会保留遍历的中间产物 安装中间包
      • go 编译注释
      • go doc 查看注释 https://godoc.org
      • 内部包 internal 只能被同一父目录的包导入
      • go list 查询可用包的信息 -json -f设置模板格式化打印
